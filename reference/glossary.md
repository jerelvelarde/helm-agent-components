# Glossary — Helm

> The cross-component vocabulary for agentic AI applications, aggregated and de-duplicated from the component research pass. See each component file's **Vocabulary** section for the term in context. For implementation terms (hooks, protocol events) see [copilotkit-primitives.md](./copilotkit-primitives.md) and [ag-ui-protocol.md](./ag-ui-protocol.md).

215 terms.

| Term | Definition |
|---|---|
| **"Thought for X seconds"** | A duration label on the collapsed trigger indicating how long the model reasoned. |
| **@-mention / #-context variable** | Typing @ or # opens a picker to attach context (files, symbols, participants, docs — including GPTs and data sources) into the prompt. |
| **Action bar / message toolbar** | Row of per-message buttons: copy, regenerate/retry, edit, thumbs up/down, read-aloud, share. |
| **Active / selected state** | The visually distinct currently-chosen tab or mode, with aria-selected and roving tab focus. |
| **active descendant / highlighted option** | The currently keyboard-focused item in the open list, tracked with aria-activedescendant while DOM focus stays on the input. |
| **Activity / progress sidebar** | Live panel listing what the agent is doing right now — queries issued, sources opened, sub-topics in progress — so the long run is observable. |
| **Activity / replay timeline** | A chronological, often scrubbable record of every command, edit, and action the agent took. |
| **Agent Card (A2A)** | JSON metadata document published by an A2A server describing its identity, capabilities, skills, endpoint, and auth requirements, letting agents discover each other and decide who is best for a task. |
| **Agents window / agent lane** | A dedicated workspace (Cursor 3.0) for running and managing many agent sessions in parallel across local, worktree, cloud, and remote environments, each session showing its task and status. |
| **Agents-as-tools** | Hierarchical pattern where a sub-agent is exposed to the orchestrator as a callable tool rather than a peer it hands off to. |
| **Always-allow / remember-this** | An option to whitelist a tool or pattern so future identical calls skip the prompt (e.g. Claude Code allow rules, GitHub Copilot 'approve TOOL for the rest of the session'). |
| **Approval / allowlist** | A gate requiring user consent before a sensitive tool (e.g., a terminal command) runs; allowlisting lets it auto-run. |
| **approval-requested state** | Tool-part lifecycle state emitted when the agent wants to run a gated tool but is waiting on the user's decision. |
| **approve / edit / reject / respond** | LangChain HITL middleware decision taxonomy for a gated tool call: run as-is, modify args first, reject with feedback, or answer an 'ask-user' tool. |
| **Approve / Reject (Allow / Deny)** | The two terminal decisions; reject usually returns control to the agent with a denial signal rather than ending the run. |
| **archive** | Soft-removal that hides a thread from the default list without deleting it; usually reachable via an includeArchived filter or an Archived view. |
| **Artifact** | A self-contained, durable output (doc, code, page, chart) rendered in its own pane that you iterate on instead of regenerating in chat — Anthropic's term for substantial, standalone content (roughly 15+ lines). |
| **Attachment / dropzone** | File/image attach via picker, drag-and-drop, or paste, usually previewed as chips above the textarea. |
| **Auto-accept / YOLO mode** | A session mode that suppresses prompts entirely; trades safety for speed (Claude Code Shift+Tab auto-accept-edits, Cursor Auto-Run/YOLO mode, --dangerously-skip-permissions). |
| **Auto-grow / autosize textarea** | The input expands with content up to a max number of rows, then becomes internally scrollable. |
| **auto-title / rename** | Thread label, frequently auto-generated by the LLM from the first message, editable inline (pencil/ellipsis menu). |
| **Avatar / digital twin** | A synthetic talking face (lip-synced, expressive) that visually represents the assistant in realtime. |
| **Barge-in / interruption** | The user starts speaking while the assistant is talking; the system cancels its in-flight audio and yields the floor. |
| **Branch / variant navigation** | The '< 2/3 >' control that pages between alternative responses created by edit or regenerate. |
| **branching / forking** | Creating an alternative path from an existing point — editing a prior message (branch / edit-and-resubmit) or regenerating spawns a sibling branch; AG-UI models this by setting parentRunId on a RUN_STARTED event for git-like lineage. |
| **capability card** | A tile describing one thing the agent can do, often clickable to prefill a prompt or open a workflow. |
| **Capability gating** | Hiding or disabling tools/actions that don't apply to the current mode (e.g., no file writes in ask mode). |
| **Captions / live transcript** | Real-time on-screen text of both sides of the spoken conversation, kept in the chat history. |
| **Chain-of-thought (CoT)** | The model's intermediate step-by-step reasoning produced before the final answer. |
| **Character / subject consistency** | Keeping the same person or object recognizable across multiple generations or edits. |
| **Checkpointer / persistence** | Durable store of graph/agent state at the interrupt so the run survives a pause (LangGraph requires a checkpointer; AG-UI emits StateSnapshot/MessagesSnapshot). |
| **Citation-forward** | Design stance (Perplexity's) where every answer is grounded and visibly cited by default rather than citing only on request. |
| **Clarifying questions** | Follow-up questions the agent asks before starting to scope the task (classic ChatGPT Deep Research opening step). |
| **Code block w/ copy** | A syntax-highlighted fenced block with a hover/persistent copy (and sometimes download) button in its header. |
| **Collapsed vs expanded** | Many tool cards default to a compact summary that can expand to show full arguments, logs, or the raw payload. |
| **Collapsible disclosure** | The expand/collapse control that hides reasoning behind a summary line by default (progressive disclosure). |
| **command palette / command menu** | The keyboard-first popover (frequently the cmdk pattern) listing commands with fuzzy search, groups, and keyboard navigation. |
| **Command(resume=…)** | LangGraph mechanism to re-enter an interrupted graph and feed the human's response back so execution continues (e.g. Command(resume={'decisions':[{'type':'approve'}]})). |
| **Composer** | The central input, usually expanding multiline, with attach, voice, model/mode selector, and tool toggles. |
| **Computer use / CUA** | A computer-using agent that perceives a screen and takes mouse/keyboard actions in a (often remote) browser/VM. |
| **Content credentials (C2PA) / SynthID** | Provenance metadata and visible/invisible watermarks marking an image as AI-generated. |
| **Control-vs-freedom spectrum** | CopilotKit's generative-UI taxonomy spans Static (UI chosen from a fixed set of hand-built components), Declarative (a structured UI specification — cards, lists, forms, widgets — is exchanged), and Open-Ended (arbitrary UI such as HTML/iframes/free-form content is passed). Any type can be developer-controlled or left to the agent. |
| **Conversation list / history rail** | The left sidebar of prior threads, often grouped by date or organized into projects/spaces. |
| **conversation starters / starter prompts** | Configured example prompts shown as clickable chips on the welcome screen (OpenAI GPTs show up to four; ChatKit/Copilot Studio calls them starter prompts, each typically a name + prompt text + optional icon). |
| **Conversational video interface (CVI)** | A framework/UX for real-time, face-to-face video chat with an AI that sees, hears, and responds. |
| **Cursor / caret** | The blinking block or shimmer appended after the last streamed token to signal the assistant is still typing. |
| **Default / preselected** | An option marked as the recommended or most-likely answer to speed the common case. |
| **Default / zero-config rendering** | Built-in fallback UI that shows a tool's name, arguments, and result generically when no custom renderer is registered (CopilotKit's 'Default Tool Rendering' / useDefaultRenderTool). |
| **Denial payload** | Structured rejection (e.g. {approved:false}) fed back to the model so it can reason about the 'no' instead of crashing. |
| **Dictation / transcribe mode** | Voice-input state that swaps the textarea for an audio recorder/waveform and transcribes to text. |
| **Dictation commands** | Spoken control words like 'new line', 'period', or 'send' interpreted as formatting/actions instead of literal text. |
| **Disambiguation / clarifying question** | An option prompt whose purpose is to resolve which of several interpretations the user meant. |
| **Disclaimer / affordance footer** | Small helper text under the input (e.g. 'AI can make mistakes', token/char counters, shortcut hints). |
| **Display-only / render-only** | A component registered purely to visualize tool args/results with no handler and no callback into the agent. |
| **Dock** | The act of pinning the panel to a viewport edge so it pushes or overlays page content rather than floating. |
| **empty result state** | The 'no matching commands' state of an open palette. |
| **Empty state / landing prompt** | The pre-conversation screen with a greeting/welcome message and suggested prompts before any messages exist. |
| **Encrypted reasoning item** | An opaque reasoning blob persisted across turns for continuity without exposing plaintext CoT (used under zero-data-retention). |
| **Endpointing** | Deciding when an utterance has ended so the segment can be finalized and sent. |
| **expiresAt / TTL** | Optional ISO-8601 deadline after which a resume is stale and produces RunError. |
| **Fan-out / fan-in** | Spawning N sub-agents in parallel (fan-out) then collapsing their results back into the lead agent (fan-in) for synthesis. |
| **Final transcript** | The committed text after the recognizer locks a segment; what lands in the composer. |
| **follow-up suggestion** | A chip generated after an assistant turn proposing the likely next question or action. |
| **Friction calibration** | Matching the weight of the confirmation (one-click vs typed-confirm vs step-up auth) to the reversibility and blast radius of the action. |
| **Full-duplex** | Listening and speaking can overlap, enabling natural interruption and backchanneling. |
| **Generation progress** | Feedback while the image renders — streamed tool-result, percentage, or a skeleton/shimmer placeholder. |
| **Generative UI / custom renderer** | Replacing the default JSON card with a bespoke React component keyed to the tool name. |
| **Generative-UI loop** | The round-trip where a rendered widget collects input and feeds it back so the agent can act on it, turning rendering into a bidirectional interaction. |
| **Grounding chunk** | In Gemini's grounding metadata, an object holding a web source's uri + title; the atomic unit a citation points at. |
| **Grounding support** | Gemini metadata mapping a text segment (startIndex/endIndex) to one or more groundingChunkIndices — the data structure that makes inline citations buildable. |
| **Handoff** | Explicit transfer of control from one agent to another while carrying conversation context; the core abstraction in OpenAI's Agents SDK / Swarm. |
| **history recall / memory** | Cross-thread retrieval where the agent references older conversations (ChatGPT full chat-history search, Claude 'Search and reference chats' / memory) vs. per-thread context. |
| **Hover preview / snippet popover** | Tooltip shown on hovering a citation that reveals the source snippet without leaving the page (popularized by Perplexity). |
| **Human-in-the-loop (HITL) tool** | A tool whose UI pauses execution to collect a user response (approve/reject/edit) before the agent continues. |
| **Image-to-image / reference** | Conditioning generation on an uploaded image for style, composition, or character consistency. |
| **incognito / temporary chat** | A conversation deliberately excluded from saved history (Claude's ghost-icon incognito chat, Gemini temporary chats). |
| **Infinite canvas** | A pan/zoom spatial board where nodes (notes, sources, generated blocks) are arranged non-linearly and the agent can add or rearrange them. |
| **Inline citation** | A numbered marker ([1], superscript, or chip) attached to a sentence that links the claim to the specific source supporting it; clicking jumps to/opens the source. |
| **Inline targeted edit** | Selecting a region of the artifact and asking the agent to change just that part rather than the whole thing (e.g., ChatGPT Canvas suggest-edits, length and reading-level controls). |
| **Inline toolbar / writing shortcuts** | Canvas's selection- and shortcut-triggered actions: for docs, adjust length, change reading level, add final polish, suggest edits; for code, add logs, add comments, fix bugs — applied to the document or a region. |
| **Inline visuals** | Images, maps, or cards rendered in the conversation while voice continues (multimodal output during voice). |
| **Inpainting** | Masking a specific region of an image and regenerating only that area from a prompt, leaving the rest intact. |
| **Interleaved thinking** | Reasoning emitted between tool calls so the model can re-plan after seeing tool results. |
| **Intermediate state streaming** | Showing the agent's progress and partial results in the canvas while it works (e.g., research notes filling in). |
| **interrupt()** | The call (e.g. LangGraph) inside a node that pauses the graph and returns a payload to the caller; its return value is whatever the human sends on resume. |
| **Launcher / toggle button** | The floating affordance (often bottom-right) that opens and closes the panel; in CopilotKit it is CopilotChatToggleButton. |
| **Lip-sync / full-duplex rendering** | Aligning generated mouth/expression with speech audio, with listening and speaking able to overlap. |
| **Live / partial transcription** | Interim, non-final words rendered in real time as the user speaks, replaced by the finalized text when the segment closes. |
| **Live Artifact** | Anthropic's Claude Cowork feature (announced Apr 2026): artifacts that stay connected to data sources/apps via MCP and refresh with current data each time they are reopened — dashboards and trackers, saved with version history. |
| **Live preview / live preview pane** | A sandboxed running render of generated code/app (Replit) that updates as the artifact changes; the pane area renders the artifact (running app, formatted doc, diagram) and re-renders on content change. |
| **Live status text** | A short, continuously updated label describing the current activity (e.g., 'Reading files…'). |
| **Live video / vision** | Continuous camera or screen feed the model interprets during a conversation, as opposed to a single uploaded image. |
| **Local vs. cloud transcription** | On-device models (privacy, offline) vs. server models (often higher accuracy, lower client cost). |
| **Markdown rendering** | Parsing assistant text as GitHub-Flavored Markdown (tables, lists, math via KaTeX, Mermaid) rather than plain text. |
| **MediaRecorder fallback** | Capture-audio-then-POST-to-a-transcription-service path used when no native browser STT exists. |
| **Message part** | A typed segment within one message (text, reasoning, tool-call, file, source-url); the AG-UI model that replaces the single-string message. |
| **Message thread / transcript** | The vertical, time-ordered list of user and assistant turns that scrolls within the main column. |
| **Mode** | A named behavior profile that changes the agent's autonomy, allowed tools, or edit permissions (e.g., ask, edit, agent). |
| **Model / effort picker** | A selector for the underlying model or reasoning depth (e.g., Auto / Fast / Thinking) usually living in the composer. |
| **new chat** | Action that creates a fresh empty thread (often labeled 'New chat' with a + or compose icon and a keyboard shortcut). |
| **Node / block** | A discrete unit on a canvas (a card, a node, a research source) that can be created, connected, or edited. |
| **onSubmitAction / submit handler** | The action fired when the form/wizard is completed, sending consolidated field values back to the agent. |
| **Open / auto-open heuristic** | The model's rule for promoting output to an artifact/canvas (substantial, self-contained, reusable) vs leaving it inline. |
| **Option / choice / candidate** | One selectable item, usually a {label, value} pair where label is shown and value is sent to the agent. |
| **Orb / waveform / persona** | The animated visual (pulsing orb, audio waveform, or character) signaling listening/thinking/speaking states. |
| **Orchestrator / lead agent** | The coordinating agent that analyzes the query, plans strategy, spawns sub-agents, and synthesizes their results (Anthropic calls it the 'LeadResearcher'). |
| **Other / free-text escape** | An 'Other…' option or an open input so the user isn't trapped when none of the choices fit. |
| **Outpainting** | Extending the image beyond its original borders, generating new content that continues the scene. |
| **Page context / readable state** | The current app data the copilot is given so its answers are grounded in what the user is looking at. |
| **Parallel interrupts** | Multiple open interrupts emitted in one run; a single resume array must address every open interrupt (mixed resolved/cancelled allowed). |
| **Partial / scoped approval** | Accepting some steps, fields, or hunks while rejecting or editing others rather than all-or-nothing. |
| **persistence / resume** | Server- or client-side storage of the event/message history so a thread survives reload, browser close, and device switch. |
| **Plan tracker** | A visible, sometimes editable list of upcoming/completed subtasks the agent intends to do. |
| **Preview-before-apply / plan review** | Showing the proposed plan or diff for inspection and edit before the agent commits the change. |
| **Progressive disclosure** | Revealing fields/steps only as they become relevant to avoid overwhelming the user. |
| **Prompt / negative prompt** | The text describing what to generate (and, in some tools, what to avoid). |
| **prompt library** | A browsable, often shareable catalog of reusable prompts/templates (Glean's AI Prompt Library, GitHub Copilot prompt files, Notion 'Draft with AI'). |
| **prompt template / saved prompt** | A reusable parametrized prompt invoked through the palette (Cursor .cursor/commands/*.md, GitHub Copilot prompt files / custom slash commands). |
| **Push vs. overlay** | Whether opening the panel reflows the page content (push) or covers it (overlay); push preserves the work surface, overlay reclaims it on narrow screens. |
| **Push-to-talk (PTT)** | Recording starts on button press/hold and stops on release or a second tap, then audio is sent for transcription. |
| **Quick reply / suggestion chip** | A tappable pill rendering a candidate answer or suggested message/action that sends or prefills on click; reduces typing and signals what the agent understands. |
| **Rationale** | The agent's explanation of why it took an action (e.g., commit rationale shown in context). |
| **reason** | AG-UI categorical routing hint for an interrupt: core values tool_call, input_required, confirmation, plus custom <framework>:<name> values. |
| **Reasoning / thinking trace** | An optional collapsible block showing the model's intermediate reasoning before the final answer. |
| **Reasoning effort / thinking budget** | A user- or developer-set level (e.g., Auto/Fast/Thinking, or a token budget) controlling how much the model reasons. |
| **Reasoning role / ReasoningMessage** | A distinct message role ("reasoning") that keeps thinking visually and structurally separate from the assistant's answer. |
| **Reasoning tokens** | Tokens the model spends thinking; billed/counted separately from visible answer tokens and often hidden by default. |
| **Remix / extend** | Deriving a new clip from an existing generation or uploaded asset rather than starting from text alone. |
| **Remix / publish / share** | Distributing an artifact via a link so others can view, use, and fork it into their own session. |
| **Render function** | The developer-supplied function/component bound to a tool that maps {args/params, status, result} to JSX shown in the thread. |
| **Report export / Page** | Exporting the finished report to PDF/doc or converting it to a shareable Perplexity Page. |
| **Research plan** | The agent's proposed outline of sub-topics/sources before it starts; in ChatGPT (GPT-5.2-based since Feb 2026) and Gemini it is shown and editable, gated behind a 'Start research' confirmation. |
| **Resize handle** | A draggable divider that lets the user widen or narrow the panel against the work surface. |
| **resolved vs cancelled** | Resume statuses: resolved = user responded (payload carries the answer, including denials); cancelled = user abandoned without meaningful input. |
| **respond / resolve callback** | The function the UI calls to send the human's decision back and unblock the paused agent (CopilotKit's useHumanInTheLoop exposes a respond callback in the Executing state; assistant-ui exposes an addResult callback). |
| **responseSchema** | JSON Schema attached to an interrupt declaring the expected resume payload; clients validate before submitting (mismatch may produce RunError). |
| **Result binding** | Connecting a tool's returned data to the props of the component that visualizes it. |
| **Resume linkage** | Binding a resume response to a specific interrupt id (AG-UI resume[].interruptId) on the same threadId. |
| **Retry** | Re-issuing a failed tool call, typically from an error state on the card. |
| **Role** | The author of the turn (user / assistant / system / tool); drives alignment, color, and avatar. |
| **Router / auto mode** | A meta-mode that lets the system pick the model/depth per turn instead of the user choosing explicitly. |
| **Run / runId** | One bounded agent execution within a thread, delimited by RUN_STARTED/RUN_FINISHED events; a thread can contain many runs. |
| **Sandbox / sandboxed iframe** | The locked-down browser context (CSP headers + iframe sandbox attribute) artifacts run in, restricting network (fetch), storage (localStorage), and host access for safety. |
| **scope / limits disclosure** | Explicit statement of what the agent can and cannot do, what data it can see, and its confidence/uncertainty, for calibrated trust. |
| **Screen share / desktop share** | User streams a window or whole desktop to the agent so it can reason about on-screen context. |
| **Search queries / webSearchQueries** | The actual queries the agent ran (surfaced in Gemini's groundingMetadata as webSearchQueries), often shown so the user can see how the answer was sourced. |
| **Search Suggestions / Search Entry Point** | In Gemini grounding, the searchEntryPoint provides HTML/CSS to render Search Suggestions chips, which Google's Terms of Service require be displayed when using Grounding with Google Search. |
| **Segmented control** | A compact set of mutually-exclusive toggle buttons used for low-cardinality mode choices. |
| **Select / mask / brush tool** | The UI for marking the exact region to edit; a tight selection yields cleaner localized edits. |
| **Selection commit** | The act (tap, or Submit for multi-select) that finalizes the choice and resumes the run; chips usually commit on tap. |
| **Semantic VAD** | Classifier that judges end-of-turn from the words spoken (likelihood the user is done), not just silence; in OpenAI's Realtime API it is tunable via an 'eagerness' setting. |
| **Send / submit button** | Primary action that dispatches the draft; commonly disabled until there is non-empty content (`canSend`). |
| **Server VAD** | Endpointing based purely on detected silence/audio volume to chunk the user's audio. |
| **Sessions view / mission control** | A panel listing all active/background agent sessions with status, used to monitor and switch between them. |
| **Shared state** | A synchronized state layer both agent and UI read from and write to in real time, keeping canvas and agent in sync. |
| **Show thinking / thinking panel** | Gemini's real-time view of what the model has learned so far and what it intends to do next while browsing. |
| **Side panel / split view / split pane** | The two-pane layout: chat as the control channel, the artifact rendered/edited beside it; a draggable divider lets the user resize the two sides. |
| **Single-select vs multi-select** | Choose-exactly-one (radio/chips) vs choose-any-subset (checkboxes), with a Submit/Confirm step for multi-select. |
| **Skeleton state** | A placeholder that mirrors the component's final shape while args/results are still streaming. |
| **skills / tools / connectors** | Named units of agent capability (functions, integrations, data sources) that may be listed so users know what's available. |
| **Slash command** | Typing '/' opens a command menu (e.g. /fix, /tests) to invoke an action without writing a full prompt. |
| **Slot / field** | One unit of data the agent needs (street, date, amount); the form maps slots to inputs and validates each. |
| **Slot filling** | Collecting the set of required parameters before the agent can proceed, conversationally or via a form. |
| **Slot value** | The specific parameter the selection fills (e.g. size, date, recipient) in a slot-filling flow. |
| **Source / citation chip** | An inline reference attached to a result card showing where the data came from (URL, document, row). |
| **Source card** | A clickable card previewing a retrieved page — title, domain + favicon, and a short snippet — usually shown in a horizontal carousel above or beside the answer. |
| **Source diversity / dedup** | Ranking that spreads citations across distinct domains rather than over-citing one site, improving trust and coverage. |
| **Source-scoping / connectors** | Limiting research to selected sites or connecting private data via MCP connectors so the report draws on the right corpus. |
| **Sources panel** | Citations or retrieved documents shown inline or in a right-side column (central to search-style answer apps). |
| **Speech-to-speech (S2S)** | A single model ingests audio and emits audio directly, preserving tone/latency vs. STT->LLM->TTS pipelines. |
| **Status indicator** | The running/spinner/done/error affordance on a tool card communicating where the call is in its lifecycle. |
| **Status machine** | The lifecycle a tool-bound component moves through, e.g. CopilotKit's InProgress -> Executing -> Complete (ToolCallStatus enum). |
| **Steering / interrupt** | Refining focus mid-run — adjusting scope or allowed sources, or interrupting with a follow-up prompt (ChatGPT added real-time controls in Feb 2026). |
| **Step (agent)** | A nested, observable phase within a run (planning, retrieval, a tool call) that can be tracked individually. |
| **Step / stage (wizard)** | One screen of a wizard; steps are ordered and often gated so step N+1 unlocks only when step N is valid. |
| **Stepper / progress indicator** | UI showing how many wizard steps exist and where the user is, with back/next navigation. |
| **Sticky mode** | Whether the chosen mode persists across turns/sessions or resets to a default each message. |
| **Stop button** | The square/stop control the send button morphs into while the agent is running, used to abort the stream. |
| **Storyboard / timeline editor** | Per-frame/per-segment prompt cards arranged on a timeline to compose or extend a generated video. |
| **Streaming / token streaming** | Progressively appending text deltas to the bubble as the model generates, instead of showing the whole reply at once. |
| **Streaming args / partial args** | Tool arguments arrive token-by-token as JSON deltas, so the component receives an incrementally-completing props object before the call is final. |
| **Streaming progress** | Real-time emission of thought summaries, text, and even inline charts/images as the research proceeds (Gemini Deep Research can natively generate charts/infographics inline, streamed as image deltas). |
| **Sub-agent / worker** | A specialized agent spawned by the orchestrator, typically with its own context window, that chases one independent thread and returns findings. |
| **Submit mode** | Configurable Enter behavior: 'enter' (Enter sends, Shift+Enter newline) vs 'ctrlEnter' (Cmd/Ctrl+Enter sends). |
| **Suggestion pills** | Tappable starter prompts or follow-ups, typically rendered above the composer or on the empty state. |
| **Summarized thinking** | A model- or system-generated paraphrase/summary of the full reasoning shown to the user instead of raw CoT (the default behavior on Claude 4 models via the Messages API). |
| **Supervisor vs. swarm** | Supervisor = central router picks the next worker each turn; swarm = routing intelligence is distributed and each agent decides independently when to hand off. |
| **Synthesis / verification phase** | Late stage where partial notes are merged into a narrative and conflicting claims are flagged and double-checked (Perplexity's pipeline). |
| **system prompt / instructions** | Backstage configuration that, in some products (e.g. Copilot Studio), can auto-generate the suggested starters and capability copy. |
| **Takeover / watch mode** | Letting the user observe the agent operate, or take manual control of the shared computer at any point. |
| **Task (A2A)** | The unit of delegated work in A2A, with a lifecycle of states (submitted, working, completed, failed, plus interrupted/terminal states) tracked across distributed agents. |
| **thread / session** | A single ordered conversation identified by a stable threadId; the unit that appears as one row in the sidebar. |
| **threadId** | Stable identifier passed to the runtime to load, persist, and resume a specific conversation. |
| **Tool call / function call** | The model's request to invoke a named tool with structured arguments; the unit that drives which component renders and with what props. |
| **Tool card** | The branded inline card (arguments + live status + eventual result) that replaces raw JSON for a tool call. |
| **Tool result / output** | The value returned from executing the tool, fed back to the model and shown in the card. |
| **Tools menu / action menu** | An explicit, clickable menu of agent tools/actions and model/persona choices attached to the input (CopilotKit's toolsMenu prop). |
| **Tracing / span** | End-to-end observability primitive that records the chain of agent calls, handoffs, and tool invocations for debugging and replay. |
| **Turn** | One user message plus its assistant response, the unit of regenerate/edit/branch operations. |
| **Turn detection / turn-taking** | Deciding when the user has finished a turn so the model should respond — via silence (server VAD) or meaning (semantic VAD). |
| **typeahead / autocomplete** | Live filtering of palette items as the user types after the trigger character. |
| **Typed confirmation / step-up** | High-friction gate where the user types a resource name or 'CONFIRM' (or re-authenticates) before an irreversible action proceeds. |
| **Typewriter / smooth streaming** | Animating streamed text character-by-character (and styling 'unterminated' tokens) so partial Markdown renders cleanly mid-stream. |
| **Undo / revert** | A control to roll back the agent's last edit or action to a prior known state. |
| **Validation** | Per-field and cross-field checks (real-time and on-submit) that block submission of malformed input. |
| **Variations / gallery / batch (n)** | Producing multiple candidates (commonly 1–4) shown as a grid for the user to pick/refine. |
| **Versioning / show changes** | Snapshots of the artifact across edits with a diff view of additions/deletions so users can compare and revert (ChatGPT Canvas's 'Show changes' covers both docs and code). |
| **View tab** | A tab that swaps the visible content of a pane (Chat / Code / Preview / Plan / Browser) rather than changing agent behavior. |
| **Voice Activity Detection (VAD)** | Logic that detects speech vs. silence to auto-start/stop capture or chunk audio without a manual button. |
| **Web Speech API (SpeechRecognition)** | Browser-native STT used directly in Chrome/Edge; not supported in Firefox/Safari, which require a server-side fallback. |
| **WebRTC session** | The peer connection commonly used to stream mic audio up and model audio down with minimal latency. |
| **Wildcard / catch-all renderer** | A single fallback renderer (name "*") for any tool call not claimed by a named renderer, so no call falls back to raw JSON. |
| **window.claude.complete** | Claude Artifacts' bridge letting JavaScript inside an artifact send a prompt to Claude and get a response back — making the artifact itself agentic; prompts are billed to the end user's own Anthropic account. |
| **workflow suggestion** | A contextual recommendation of a multi-step action relevant to the current document/state (e.g. Harvey surfacing workflow suggestions inside a thread). |
| **workspace grouping** | Higher-level containers that bundle related threads with shared context — Claude Projects, Perplexity Spaces (formerly Collections), Harvey Vaults. |
